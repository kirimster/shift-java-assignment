# Утилита фильтрации содержимого файлов

Java-утилита для фильтрации данных из текстовых файлов с разделением по типам (целые числа, вещественные числа, строки).

## Возможности

- **Фильтрация данных**: автоматическое определение типа каждой строки
- **Гибкий вывод**: настройка пути, префикса и режима записи файлов
- **Статистика**: краткая и полная статистика по обработанным данным
- **Отказоустойчивость**: продолжает работу при ошибках в отдельных файлах

## Требования

- **Java**: версия 8 или выше
- **Система сборки**: используется чистый Java

## Установка и запуск

### 1. Сборка из исходного кода

```bash
# Клонирование репозитория (если есть)
git clone https://github.com/kirimster/shift-java-assignment
cd Task

# Компиляция всех Java-файлов
javac *.java
```
### 2. Запуск программы
Базовый запуск
```
java Main file1.txt file2.txt
```
С опциями:
```
java Main -o ./output -p result_ -s -a input1.txt input2.txt
```
### 3. Сборка в JAR-файл
```
# Компиляция
javac *.java

# Создание JAR с указанием главного класса
jar cfe util.jar Main *.class

# Запуск JAR
java -jar util.jar [опции] файлы...
```
### Использование
#### Синтаксис команды
```
java Main [ОПЦИИ] ФАЙЛ1 ФАЙЛ2 ...
```
#### Основные опции
```
-o <путь>	    Путь для выходных файлов (по умолчанию стоит текущая папка (.))
-p <префикс>	Префикс имен выходных файлов (по умолчанию без префикса)
-a	            Режим добавления в существующие файлы	(перезапись файлов)
-s	            Краткая статистика (только количество)	(без статистики)
-f	            Полная статистика (min/max/sum/avg)	(без статистики)
```
#### Примеры использования
Пример 1: Базовый запуск с двумя файлами
```
java Main data1.txt data2.txt
```
Пример 2: Запуск с полной статистикой и префиксом
```
java Main -f -p analysis_ input.txt
```
Пример 3: Запуск из условия задания
```
java -jar util.jar -s -a -p sample- in1.txt in2.txt
```
Создаст файлы: sample-integers.txt, sample-floats.txt, sample-strings.txt
### Технические детали реализации
#### Версии и окружение
- Java: версия 21.0.7 (OpenJDK Temurin)

- Минимальная требуемая версия: Java 8 (программа использует только стандартную библиотеку)

- Система сборки: чистый Java (компиляция через javac, без Maven/Gradle)

- Сторонние библиотеки: не используются

### Статистика
#### Краткая статистика (-s)
```
# Пример
Целые числа: Количество: 3
Вещественные: Количество: 3  
Строки: Количество: 5
```
#### Полная статистика (-f)
```
# Пример
Целые числа: Количество: 3, Min: 45, Max: 1234567890123456789, Sum: 1234567890123456734, Average: 411522630041152244.67
Вещественные: Количество: 3, Min: -0.0010, Max: 3.1416, Sum: 3.1405, Average: 1.0468
Строки: Количество: 5, Min длина: 4, Max длина: 47
```
### Структура проекта
```

├── src/
│   ├── Main.java                 # Точка входа
│   ├── ArgumentParser.java       # Парсер аргументов
│   ├── AppConfig.java            # Хранилище настроек
│   ├── FileLineReader.java       # Чтение файлов
│   ├── DataTypeDetector.java     # Определение типа данных
│   ├── DataProcessor.java        # Основная логика
│   ├── StatisticCollector.java   # Сбор статистики
│   └── FileLineWriter.java       # Запись в файлы
├── input/                        # Примеры входных файлов
│   ├── in1.txt
│   └── in2.txt
├── output/                       # Папка для выходных файлов
├── README.md                     # Эта инструкция
└── util.jar                      # Готовый JAR-файл
```
### Дополнение
Если вы запускаете программу из корневой папки проекта, вам нужно указывать правильные пути к входным файлам и выходной папке. Например, при структуре, указанной выше, команда из примера задания будет выглядеть так: 
```
java -jar util.jar -o output -s -a -p sample- input/in1.txt input/in2.txt
```
Тут также просто файлы переходят в папку output
